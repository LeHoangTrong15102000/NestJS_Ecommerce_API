## 🎯 PHÂN TÍCH CHI TIẾT CÁC MODULE QUAN TRỌNG

### 1. 🔴 **WISHLIST MODULE** (Ưu Tiên Cao Nhất)

**Tại sao cần test:**
- ✅ Module mới được implement gần đây
- ✅ Có background jobs (price check, alerts) với BullMQ
- ✅ Có cron job chạy hàng ngày
- ✅ Business logic phức tạp (price alerts, collections, notifications)
- ✅ Tích hợp với Cart module

**Các tính năng cần test:**
- Add/Update/Remove wishlist items
- Get wishlist items (pagination, filtering, sorting)
- Move item to cart
- Price alert system (set target price, check price drops)
- Collection management (create, update, delete, add/remove items)
- Background jobs (price check, send email alerts)
- Cache management

**Ước tính: 95-115 test cases**

---

### 2. 🔴 **AI ASSISTANT MODULE** (Ưu Tiên Cao)

**Tại sao cần test:**
- ✅ Module mới, tích hợp với Anthropic Claude API
- ✅ Critical cho user experience
- ✅ Có rate limiting và quota management
- ✅ Error handling phức tạp (quota exceeded, timeout, auth failed)

**Các tính năng cần test:**
- Chat với AI (streaming & non-streaming)
- System prompt configuration
- Rate limiting per user
- Error handling và fallback mechanism
- Token usage tracking
- Conversation history management

**Ước tính: 60-80 test cases**

---

### 3. 🔴 **PAYMENT MODULE** (Critical)

**Tại sao cần test:**
- ✅ Critical business logic - Financial transactions
- ✅ Tích hợp với Stripe
- ✅ Webhook handling
- ✅ Background job processing
- ✅ Refund processing

**Các tính năng cần test:**
- Create payment intent
- Process payment
- Handle webhook events (payment succeeded, failed, refund)
- Payment status updates
- Queue processing

**Ước tính: 80-100 test cases**

---

### 4. 🔴 **USER MODULE** (Core Business)

**Tại sao cần test:**
- ✅ Core business logic
- ✅ RBAC integration (Role-Based Access Control)
- ✅ Password hashing
- ✅ Complex permission checks
- ✅ Prevent self-update/delete

**Các tính năng cần test:**
- List users (pagination, filtering)
- Create user (with role assignment, password hashing)
- Update user (RBAC checks, prevent admin update by non-admin)
- Delete user (soft delete, RBAC checks)
- Verify role permissions

**Ước tính: 70-90 test cases**

pnpm test:cov

# Xem coverage report
open coverage/lcov-report/index.html

# Watch mode
pnpm test:watch
