## 1. Tổng Quan Về Supabase

### 1.1 Supabase Là Gì?

**Supabase** là một nền tảng Backend-as-a-Service (BaaS) mã nguồn mở, được xây dựng trên nền tảng **PostgreSQL**. Supabase cung cấp một bộ công cụ hoàn chỉnh bao gồm:

| Tính năng | Mô tả |
|-----------|-------|
| **PostgreSQL Database** | Database quan hệ mạnh mẽ với đầy đủ tính năng ACID |
| **Authentication** | Hệ thống xác thực tích hợp (Email, OAuth, Magic Link) |
| **Storage** | Lưu trữ file với CDN tích hợp |
| **Realtime** | WebSocket subscriptions cho real-time updates |
| **Edge Functions** | Serverless functions chạy gần người dùng |
| **Auto-generated APIs** | REST và GraphQL APIs tự động từ schema |

### 1.2 Các Phương Thức Kết Nối Database

Supabase cung cấp **3 phương thức kết nối** chính:

#### 🔹 Direct Connection (Port 5432)
```
postgresql://postgres:[PASSWORD]@db.[project-ref].supabase.co:5432/postgres
```
- **Ưu điểm**: Latency thấp nhất, không có overhead
- **Nhược điểm**: Yêu cầu IPv6 (IPv4 là add-on trả phí)
- **Phù hợp**: VPS, containers chạy lâu dài, migrations

#### 🔹 Session Mode Pooling (Port 5432 qua Supavisor)
```
postgres://postgres.[project-ref]:[PASSWORD]@aws-0-[region].pooler.supabase.com:5432/postgres
```
- **Ưu điểm**: Hỗ trợ cả IPv4 và IPv6
- **Nhược điểm**: Có overhead từ pooler
- **Phù hợp**: Servers cần IPv4

#### 🔹 Transaction Mode Pooling (Port 6543)
```
postgres://postgres:[PASSWORD]@db.[project-ref].supabase.co:6543/postgres
```
- **Ưu điểm**: Tối ưu cho serverless, auto-scaling
- **Nhược điểm**: **KHÔNG hỗ trợ prepared statements**
- **Phù hợp**: Vercel, AWS Lambda, Edge Functions

### 1.3 PostgreSQL Version & Extensions

- **PostgreSQL Version**: 15.x (tự động cập nhật)
- **Extensions hỗ trợ**: 90+ extensions bao gồm:
  - `PostGIS` - Geographic queries
  - `pgvector` - AI/ML embeddings
  - `pg_cron` - Scheduled jobs
  - `pg_stat_statements` - Query monitoring
  - `uuid-ossp` - UUID generation

## 6. Hướng Dẫn Tích Hợp Chi Tiết

### 6.1 Bước 1: Tạo Project Trên Supabase

1. Truy cập [https://supabase.com](https://supabase.com)
2. Đăng ký/Đăng nhập
3. Click **"New Project"**
4. Chọn:
   - **Organization**: Tạo mới hoặc chọn có sẵn
   - **Project name**: `nestjs-ecommerce-dev` (cho dev) hoặc `nestjs-ecommerce-prod` (cho prod)
   - **Database Password**: Tạo password mạnh (LƯU LẠI!)
   - **Region**: `Southeast Asia (Singapore)` - gần Việt Nam nhất
5. Click **"Create new project"**
6. Đợi 1-2 phút để project được tạo

### 6.2 Bước 2: Lấy Connection String

1. Vào **Project Settings** → **Database**
2. Scroll xuống **Connection string**
3. Chọn tab **URI**
4. Copy 2 connection strings:

**Transaction Mode (cho Prisma Client runtime):**
```
postgres://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres
```

**Session Mode (cho Prisma Migrate):**
```
postgres://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres
```

### 6.3 Bước 3: Cập Nhật Environment Variables

Mở file `.env` và cập nhật:

```env
# ===== SUPABASE DATABASE CONFIGURATION =====

# Transaction mode - dùng cho Prisma Client (runtime queries)
# QUAN TRỌNG: Thêm ?pgbouncer=true để disable prepared statements
DATABASE_URL="postgres://postgres.[PROJECT-REF]:[YOUR-PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres?pgbouncer=true"

# Session mode - dùng cho Prisma Migrate (migrations, introspection)
DIRECT_URL="postgres://postgres.[PROJECT-REF]:[YOUR-PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres"
```

### 6.4 Bước 4: Cập Nhật Prisma Schema

Mở `prisma/schema.prisma` và cập nhật datasource:

```prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")  // 👈 THÊM DÒNG NÀY
}
```

### 6.5 Bước 5: Chạy Migration

```bash
# Generate Prisma Client
npx prisma generate

# Push schema lên Supabase (cho development)
npx prisma db push

# HOẶC chạy migrations (cho production)
npx prisma migrate deploy
```

### 6.6 Bước 6: Seed Data (Tùy chọn)

```bash
# Chạy seed script nếu có
npx prisma db seed
```

### 6.7 Bước 7: Verify Kết Nối

```bash
# Mở Prisma Studio để kiểm tra
npx prisma studio
```

Hoặc chạy app và test:
```bash
pnpm run start:dev
```
